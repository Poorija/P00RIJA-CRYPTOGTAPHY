      خروجی گرفتن از اطلاعات

      از این بخش میتوانید برای زمانی که قصد تغییر سیستم خود را دارید استفاده کنید. تمامی کلیدهای خصوصی و عمومی، رمزهای عبور تولید شده و تنظیمات برنامه به همراه کلید مستر خروجی گرفته میشود.

      وارد کردن اطلاعات

      فایل پشتیبان (.poorija-backup) را وارد کنید. توجه داشته باشید که این کار اطلاعات فعلی برنامه را بازنویسی میکند و به رمز عبوری که در هنگام خروجی گرفتن ثبت کردید نیاز دارد.

      ایجاد پیام خودتخریب

      پیغامی ایجاد کنید که پس از گذشت زمان معین یا تعداد دفعات مشاهده مشخص، از بین برود.

      ساعت (0-23)

      دقیقه (0-59)

      ثانیه (0-59)

      مقدار 0 برای همه یعنی بدون محدودیت زمانی.

      هر بار بازگشایی پیغام، یک بار محاسبه می‌شود. عدد 0 یعنی بدون محدودیت دفعات.

      استخراج متن از تصویر

      امحای امن فایل (File Shredder)

      این ابزار با استفاده از File System Access API محتوای فایل اصلی شما را در هارد دیسک با صفر (0x00) بازنویسی (Overwrite) کرده و سپس آن را خالی میکند تا به صورت امن امحا شده و غیرقابل بازیابی شود.

      راهنمای File System Access API:

      • Google Chrome / Edge: این قابلیت به صورت پیش‌فرض فعال است. اگر کار نکرد، به chrome://flags یا edge://flags بروید و File System Access API را فعال (Enable) کنید.
      • Firefox / Safari: این مرورگرها هنوز پشتیبانی کاملی از این API ندارند.

      توجه بسیار مهم: پس از پایان عملیات صفرنویسی (Shredding)، فایل شما خالی می‌شود اما از روی هارد حذف نمی‌شود. برای امحای نهایی، شما باید فایل خالی شده را به صورت دستی (Shift+Delete) از روی هارد پاک کنید.

      قابلیت‌های جدید نسخه 2.80

      • Passkey / Biometric Unlock: باز کردن سریع برنامه با WebAuthn و passkey روی دستگاه‌های پشتیبانی‌شده.
      • Secure Share: ساخت لینک یا باندل امن برای متن، یادداشت، فایل کوچک و کلید عمومی با انقضا و محدودیت مشاهده.
      • Digital Signatures: امضا و راستی‌آزمایی متن و فایل با ECDSA و RSA-PSS.
      • Security Center: پایش سلامت امنیتی برای passkey، 2FA، بکاپ، رمزهای ضعیف و کلیدهای legacy.
      • Smart Wizard & Secure Notes: سناریوهای آماده برای کارهای رایج به‌همراه یادداشت‌های امن قابل اشتراک.
      • Secure Chat: چت امن با طراحی موبایل‌محور، رله رمزنگاری‌شده، TURN، Web Push، پیام صوتی، پیام زمان‌دار، تیک‌ها و تماس صوتی/تصویری.
      • PWA + Desktop: نصب‌پذیری وب، service worker، داشبورد واکنش‌گرا و بسته‌بندی دسکتاپ Tauri.

      راهنمای چت امن

      • اتصال: Server URL همان دامنه HTTPS برنامه است. اگر Docker با profile TURN اجرا شده باشد، برنامه TURN URL/User/Password را از `/turn-config` می‌گیرد.
      • ارسال پیام: هنگام اولین ارسال، برنامه سشن RSA -> AES-GCM را خودکار می‌سازد؛ دیگر لازم نیست کاربر جداگانه دکمه کلید را بزند.
      • آفلاین: رله فقط payload رمزنگاری‌شده و اعلان عمومی Push را نگه می‌دارد؛ متن پیام روی سرور قابل خواندن نیست.
      • تماس‌ها: برای اینترنت واقعی TURN را با دامنه/IP عمومی تنظیم کنید. قطع لحظه‌ای ICE فوراً تماس را نمی‌بندد و فقط در حالت failed/closed پایان می‌دهد.

      گردش‌کارهای پیشنهادی

      • برای خودم رمزنگاری می‌کنم: از Wizard گزینه Encrypt for Myself را بزنید تا AES-256-GCM و password mode آماده شود.
      • برای شخص دیگر می‌فرستم: از Secure Share یا File Encrypt با RSA-OAEP-3072/4096 و کلید عمومی گیرنده استفاده کنید.
      • اصالت محتوا مهم است: در تب Digital Signatures امضای متن/فایل بسازید و سمت گیرنده verify کنید.
      • می‌خواهم ریسک‌ها را ببینم: تب Security Center را باز کنید تا وضعیت passkey، 2FA، بکاپ و legacy records را یک‌جا ببینید.

      AES-256-GCM
